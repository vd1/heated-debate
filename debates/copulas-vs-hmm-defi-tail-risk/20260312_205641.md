**Critical Findings (highest risk first)**

Regime-conditional t-copulas need enough crisis observations per regime; DeFi rarely gives that cleanly. You risk precise-looking but unstable tail parameters.

2. **State model and PD factors are likely endogenously entangled.**

4. **Governance burden is understated.**  
Two-speed calibration + Bayesian pooling + scenario overlays creates many degrees of freedom and weak auditability about “what actually drove score changes.”

5. **Cross-protocol shrinkage can hide idiosyncratic failure modes.**  
Pooling stabilizes estimates but can wash out protocol-specific smart-contract/oracle risks, which are often the true tail drivers.

**Architectural Challenge to the Premise**

- Keeping the 5 independent PD components and adding a dependence meta-layer may preserve legacy structure, not truth.  
- In DeFi, loss propagation is often networked (liquidation cascades, bridge bottlenecks), so dependence is structural, not just statistical.

**Stronger alternative framing**

- Make **distributionally robust dependence bounds** the production core (capital/limits), not just guardrails.  
- Use a lightweight online regime detector only for **control-plane actions** (throttles, margins), while tail aggregation remains worst-case bounded.  
- Treat copulas as an offline scenario/explainer tool until tail data depth materially improves.

**Bottom line**

- The proposed hybrid is directionally smart but likely too parameter-heavy for current data reality.

**Drop-In Governance Spec (v0.1)**

1. **Ambiguity set (explicit, constrained)**
- Define admissible dependence at time `t` as `A_t = {C_t}` such that:
- Marginals: each component PD stays within calibration band  
`p_i(t) ∈ [p̂_i(t)-δ_i, p̂_i(t)+δ_i]`.
- Pairwise rank correlation:  
`ρ_ij(t) ∈ [ρ_ij^L, ρ_ij^U]`.
- Tail dependence:  
`λ_ij^U(t) ∈ [λ_ij^L, λ_ij^U]`.
- Topology constraints: stronger bounds only for structurally linked pairs (shared oracle, collateral route, bridge); weak/near-zero caps for unlinked pairs.
- Time smoothness:  
`d(C_t, C_{t-1}) ≤ b_step`, plus daily budget  
`Σ_t d(C_t, C_{t-1}) ≤ B_day`.

2. **Core output**
- Compute interval:  
`PD_low(t) = inf_{C∈A_t} PD(C)`  
`PD_high(t) = sup_{C∈A_t} PD(C)`.
- Use interval width `W(t)=PD_high-PD_low` as uncertainty signal, not just level.

3. **Feature firewall (anti-double-counting)**
- Core aggregation inputs: only five PD component outputs + structural graph stats.
- Regime controller inputs: market microstructure + operations telemetry.
- Governance rule: no feature overlap allowed.
- CI checks: set intersection test + max allowed correlation/MI between controller features and PD component residuals.

4. **Deterministic policy map**
- Policy key is `(state, PD_high bucket, W bucket)`.
- Example buckets:  
`PD_high`: `<1%`, `1–3%`, `3–7%`, `>7%`  
`W`: `<0.5%`, `0.5–2%`, `>2%`.
- Actions (predefined deltas): margin `+0/+5/+10/+20%`, LTV cap `0/-2/-5/-10 pts`, liquidity haircut `+0/+3/+7/+15%`.
- Transition controls: hysteresis, minimum dwell time, max daily action change, emergency override triggers.

5. **Budgeted adversarial search**
- Optimize worst-case loss over `A_t`, but with persistence penalty:
`max_{C_t∈A_t} Loss(u_t, C_t) - η d(C_t,C_{t-1})`.
- Add control inertia penalty to avoid flip-flopping:
`-κ ||u_t-u_{t-1}||`.
- Enforce hard move budgets (`b_step`, `B_day`) in solver constraints.

6. **Promotion gates for Phase 3**
- Policy turnover below threshold.
- False-positive throttle rate below threshold.
- Replay reproducibility (same inputs => same outputs).
- Stress replay tail coverage meets target.
- Only promote copula/HMM coupling if all gates pass for a full evaluation window.

**Key tradeoff**
- Tighter ambiguity constraints reduce conservatism but increase miss risk; looser constraints improve safety but can over-throttle. The control discipline above is what keeps robust bounds usable in production.

- Fréchet-Hoeffding consistency for each pair via copula bounds used by the solver.
- Tail-correlation compatibility via calibrated envelopes:
  `lambda_ij in [L_ij(rho_ij), U_ij(rho_ij)]`
  where `L_ij, U_ij` are pre-fit from feasible copula family samples.
- Topology mask constraints: tighter bounds only on structurally linked pairs; capped weak dependence otherwise.

If constraints are infeasible, relax in fixed order with logged deltas: topology caps -> tail bounds -> correlation bounds.

2. **Distance metric (locked)**
Use:
`d(C_t,C_{t-1}) = sqrt(sum_{i<j} w_rho,ij*(Delta rho_ij)^2 + w_lambda,ij*(Delta lambda_ij)^2)`

Units: dependence-points per step (dimensionless).
Controls:
- Step budget: `d <= b_step`
- Daily budget: `sum d <= B_day`

3. **Incident override state (hard jumps)**
Add deterministic `INCIDENT` state for predeclared classes:
- `smart_contract_exploit`
- `oracle_halt`
- `bridge_pause`

In `INCIDENT`:
- Smoothness budgets are bypassed for affected dependency pairs.
- Policy actions switch to precommitted emergency floor/ceiling values.
- Exit requires incident-clear signal plus minimum dwell.

4. **Real-time SLA and fallback hierarchy**

- Per-block scoring: `<= 150 ms`
- Hourly exact recompute: `<= 30 s`

- `M0` hourly exact `inf/sup` over full constrained set.
- `M1` per-block approximation (linearized around last exact solution).
- `M2` timeout fallback: last-good bounds for up to `N` blocks, then fail-safe conservative cap.

All mode switches are logged and replayable.

- `PD_low(t) = inf_{C in A_t} PD(C)`
- `PD_high(t) = sup_{C in A_t} PD(C)`
- `W_dep(t) = PD_high - PD_low`

- `W_dep`: dependence ambiguity
- `W_miss`: data-missingness uncertainty
- `data_quality in {good, degraded, bad}`

Policy uses `W_dep` directly; `W_miss` is handled by `data_quality` branch (not merged blindly into `W_dep`).

- `margin_add2 >= margin_add1`
- `ltv_cut2 >= ltv_cut1` (cut magnitude)
- `haircut_add2 >= haircut_add1`

Also enforce bucket continuity limits and daily max delta constraints.

Feature sets must be disjoint by construction.

- Regress controller features and PD components on market-factor controls.
- Compute conditional dependence on residuals (CMI/permutation test).
