# Motion

A third pathfinder route built on dense embeddings ("PF3") can REPLACE the
paper-vs-paper recourse to per-candidate agentic drafting that PF1 and PF2 both
depend on, without losing the shared-artefact discipline that makes their
verdicts trustworthy.

## The debate must resolve which of these is true

1. **Replacement (the strong motion).** Embedding similarity, calibrated against
   a gold set, can decide "plausible vs not-applicable" well enough to retire
   per-candidate LLM drafting. PF3 stands alone.
2. **Prefilter (the weak claim).** Embeddings cut the candidate count to a
   sub-quadratic k-nearest-neighbour shortlist, but an LLM agent must still draft
   the verdict for each survivor. The quadratic cost becomes sub-quadratic; the
   agentic verdict does not go away.
3. **Orthogonal discovery (a different claim).** Embeddings surface joins the
   hand-built vocabularies cannot name, but neither replace nor prefilter the
   existing routes; they feed the vocabulary, not the verdict.

Do not let the debate settle into vague agreement. The crux is sharp: similarity
is not a shared artefact. A useful outcome names the experiment that would
decide between 1, 2 and 3, and states the calibration or hybrid design that the
strong motion would require to hold.

Full architecture and the empirical collision evidence are in the attached
context.

# Question the premise

- The agent is not magic; it is a structured prompt over a pattern matcher. The structure (artefact / role / scope / disqualifier) is doing the work, not the parameters.
- "Similarity is not a shared artefact" is true of *what you chose to embed* (free text). Embed the artefact tuple itself and similarity becomes a shared-artefact test by construction.
- The 102 collisions are a labelled adversarial set most ML teams would pay for. Treating them as a defeat for vectors inverts the usual relationship between gold sets and learned similarity.

# The radical PF3 design

Stop embedding papers. Embed **claim-shapes**.

- One-time structured extraction per paper and per edge: `(artefact, role ∈ {targets, discusses, benchmarks-against, cites, dequantises, simulates-classically}, scope, modality)`. Done once with a cheap model, cached, byte-stable.
- Bi-encoder over claim-shapes. Cosine is now: "do these two tuples name the same artefact in compatible roles?"
- Role mismatch (`discusses` vs `targets`, `benchmarks-against` vs `executes`) is encoded as a hard mask, not a soft similarity. The adiabatic-in-citation and boson-sampling-aside cases are killed at the mask layer, not the cosine layer.
- Contrastive fine-tune the encoder on the existing 6/102 gold set with leave-one-paper-out folds. This is the textbook regime for contrastive learning: tiny positive class, large hard-negative class, all hard negatives are already labelled.

That is replacement, not prefilter. No per-candidate agent at query time.

# Calibration spec (what would let the motion stand)

- Family-stratified thresholds. Crypto, QEC, networking, assurance, primitives each get their own similarity geometry; one global cut will fail.
- Conformal prediction wrapper: emit `plausible` only when family-conditional p-value clears a target FDR (proposal: 5% on held-out folds).
- Pass bar to retire the agent: on held-out folds of the 108-pair set, **precision >= 95% at recall >= 80%**, with zero false positives on the four canonical traps (review paper, citation-only token, classical simulator consuming circuits, "for example" aside).
- Soft bar (prefilter falls out automatically): if precision plateaus below ~70%, Motion 2 wins and the agent stays on survivors. Either way the quadratic blowup dies.

# The experiment that decides 1 vs 2 vs 3

Run four arms on the 108-pair gold, leave-one-paper-out:

1. Naive cosine over full paper text vs edge text.
2. Cosine over curated capability quotes vs edge text.
3. Claim-shape bi-encoder, role-masked, contrastively tuned on folds.
4. Claim-shape + cross-encoder reranker (tiny, distilled from the agent's existing verdicts and disqualifiers).

Decision rule:

- Arm 3 or 4 clears the hard bar → **Motion 1**.
- No arm clears the hard bar but arm 3 keeps recall >= 95% at any precision → **Motion 2**, with arm 3 as the prefilter.
- Any arm surfaces *new* true positives outside the 6 (validated by a one-shot agent audit on the new candidates) → **Motion 3 holds in addition**, and PF3 ships as a vocabulary-feeder regardless of 1 vs 2.

This is one experiment, three possible verdicts, all falsifiable.

# Unconventional moves the opposition will not have considered

- **Distil the agent into the metric.** Run the existing PF1 agent over a stratified 2-3k pair sample, harvest the `(named-artefact, disqualifier)` strings, embed those, and use the embedded *disqualifier space* as a veto layer. The agent's reasoning becomes a frozen lookup; the "agentic IQ" is now a vector op.
- **Adversarial synthesis.** Use the agent once to generate hard negatives (review papers, citation-only mentions, classical-simulator-of-circuits) and train against them explicitly. The 102 collisions become a starting point, not a ceiling.
- **Role-tag at extraction, not at scoring.** Sentence-level discourse-role tagging during extraction is the cheapest way to encode "targets vs discusses" and is the single intervention most likely to clear the hard precision bar.
- **Drop embeddings entirely if needed.** If claim-shape extraction is good enough, a logistic classifier over tuple features may beat the bi-encoder. PF3's real bet is structured extraction; the vector store is the cheap consequence.

- Contrastive tuning on 108 pairs is small; leave-one-paper-out plus synthetic hard negatives is the answer, and the experiment will tell us if it generalises.

- Ship role-masked bi-encoder + conformal thresholds as PF3.
- Gate retirement of the per-candidate agent on the hard bar above, per family.
- If the bar is missed, the same artefact (claim-shape index) is exactly the prefilter Motion 2 wants, at zero extra cost. The strong motion fails gracefully into the weak claim. There is no path where building PF3 this way is wasted.

**Main Objections**

2. **Shared artefact discipline requires evidence, not labels.**

3. **The 108-pair gold set is too small and too biased.**

4. **Synthetic hard negatives are dangerous calibration theater.**

5. **Distilling disqualifiers into vectors loses the point of disqualifiers.**  
   A disqualifier is useful because it is inspectable: “this is only citation context,” “this is a classical simulator,” “this benchmark consumes but does not execute.” A vector veto may correlate with those cases but cannot explain a novel one unless it emits evidence.

6. **Conformal prediction does not rescue bad labels.**  
   Conformal wrappers calibrate uncertainty under exchangeability. They do not guarantee semantic correctness, especially under new paper families, new vendor modalities, or distribution shift.

**The Strong Motion’s Real Requirement**

For Motion 1 to hold, PF3 must produce an applicability certificate, not a similarity score:

```text

edge span:
shared artefact:
paper role:
edge role:
compatibility rule:
explicit disqualifier checked:
verdict:
calibration confidence:
```

If PF3 cannot emit that certificate with cited spans, it has not replaced the shared-artefact discipline. It has only replaced readable reasoning with a latent score.

**Better Experiment**

The proposed four-arm test is a start, but it is too forgiving. The deciding experiment should be:

1. Train only on existing drafted pairs.
2. Hold out by paper family and by time, not just by paper.
3. Include PF1 candidates, PF2 candidates, and embedding-discovered candidates.
4. Require span-level artefact extraction accuracy before verdict scoring.
5. Score three things separately: candidate recall, verdict precision, and certificate validity.
6. Have blind human or agent adjudication only after PF3 commits its certificate.

Decision rule:

- **Motion 1 wins** only if PF3 reaches at least 95% precision, at least 80% recall, calibrated per family, and emits valid evidence certificates for every `plausible`.
- **Motion 2 wins** if PF3 has high recall but needs an agent or verifier to write or validate the certificate.
- **Motion 3 wins** if PF3 mainly discovers new vocabulary, artefact aliases, or cross-family joins that the bridge missed.

My prediction: claim-shape PF3 lands in Motion 2 with some Motion 3 value. It will be an excellent candidate generator and vocabulary miner, but not a verdict replacement.

**Radical Alternative**

Drop “embedding verdicts” entirely. Build a typed applicability calculus:

- Extract evidence spans.
- Normalize artefacts into a typed ontology.
- Use hard compatibility rules for roles and modalities.
- Use embeddings only for alias discovery and fuzzy ontology expansion.
- Require an LLM or symbolic verifier only when rules cannot decide.

That keeps the anti-hype discipline intact while still killing much of the quadratic cost. PF3 should become evidence-carrying retrieval, not similarity pretending to be judgment.
