### Agent B · `gpt-5.5` · 37s

Counterposition: this proposal is too vulnerable to measuring transcript management, not an internal search map.

**Main Weaknesses**

The “interruption and resume” test mostly probes context dependence. If you interrupt and restart from edited text, you are not testing whether the original run had a persistent map; you are creating a new forward pass over a new prompt. Any “resistance” to false branches may just be mathematical competence or textual contradiction detection.

The unit-distance trace is a dangerous flagship specimen. It is rewritten, post-hoc, human-digested, and now public. Its branch graph may reflect exposition, not the model’s actual search topology. Treating that trace as ground truth risks testing whether models understand a polished narrative of discovery.

Hidden-state probes are weak evidence. Linear probes can decode many things the model does not causally use. “Hidden beats text embedding” is also too low a bar unless the text baseline uses equally strong access to context. Otherwise the experiment just proves activations encode the text better than your text embedding.

Branch-label prediction is confounded by rhetorical cues: “maybe,” “this fails,” “tempting,” “but,” “the catch is.” A probe may learn literary markers of mathematical backtracking, not a map.

Lesioning is seductive but underspecified. If perturbing a “dead branch” direction makes the model worse, that does not prove a branch map. It may just damage uncertainty, negation, arithmetic checking, or local coherence.

**Sharper Failure Modes**

A model could pass these tests with no persistent tree-like representation: it may simply perform strong local reconstruction from context.

A model could fail these tests despite having useful internal search state, because your interruptions destroy the state and replace it with edited text.

A model could contain many shallow branch traces without a navigable “map.” The proposal needs to distinguish cached semantic residue from an active control structure.

**Better Experiment**

Use controlled synthetic proof-search environments with known ground-truth trees. Give the model problems where branches, dead ends, hidden dependencies, and live frontiers are generated by construction, not inferred from a human narrative.

Then test three stronger claims:

1. **Invariance:** Does the same latent branch state appear under paraphrased traces with different wording?
2. **Causality:** Can activation patching from a state where branch A is live into a state where branch B is live selectively redirect continuation?
3. **Counterfactual specificity:** Can you alter one branch-status variable while preserving local fluency and mathematical competence?

Most important: compare against a “transcript-only reconstruction” agent with equal visible context but no original KV/cache state. If the supposed map does not outperform reconstruction under matched compute and context, it is not a map in the interesting sense.

**Radical Alternative**

Stop using natural language CoT as the primary object. Instrument an external search sandbox: theorem prover, proof puzzle, codebase bug hunt, or finite combinatorial construction. Let the model make explicit branch actions. Record actual explored nodes. Then ask whether internal activations encode the external search graph better than the written transcript does.

The current proposal is directionally good, but too transcript-centric. The clean hypothesis should be: “There exists a causally active latent variable representing frontier state, invariant across paraphrase and patchable across runs.” Anything weaker risks redescribing ordinary context processing as an MCTS-like map.

    - suffix-only probes,
    - transcript parsers,
    - learned finite-state trackers,
    - explicit graph-state baselines,
    - future-token baselines.

- **Causal Interchange**
  - Use paired runs differing only in one target variable, e.g. `A7Q live` vs `A7Q dead`.
  - Patch only the inferred target-variable component.
  - Measure whether branch choice changes as if that variable changed.
  - Decode non-target variables post-patch to verify stability.

- **Metrics**
  - Target action logit/probability shift.
  - KL drift over non-target actions.
  - Syntax/action validity.
  - Task success.
  - Stability of decoded non-target branch states.
  - Robustness under node renaming and encoding changes.

- **Controls**
  - Random subspace patching.
  - Full activation patching as an upper-bound sanity check.
  - Suffix-only histories.
  - Explicit graph-state model.
  - Learned finite-state transcript tracker.
  - Randomized rhetorical/token-position cues.
  - Action-menu order randomization.

- **Success Gate**
  - Predictive accuracy beats strong transcript/state-tracker baselines.
  - Representation transfers across renamings and encodings.
  - Interchange causes selective branch redirection.
  - Non-target drift remains low.
  - Syntax and task competence remain intact.
  - Effects do not depend on labels, rhetoric, or menu position.

- **Falsifiers**
  - Probe collapses under renaming or encoding changes.
  - Patch changes confidence/style rather than branch selection.
  - Non-target branch variables drift substantially.
  - Random subspaces work as well as learned components.
  - Effects are explained by suffix/action-menu statistics.

- **Staging**
  1. Toy symbolic search with exact state.
  2. Logged theorem-prover or code-search tasks.
  3. Natural math traces, including unit-distance, only as qualitative transfer tests.

**Interpretation Rule**

A finite-state parser matching predictive accuracy does not falsify state use; it only shows the state is transcript-derivable. The decisive evidence is selective causal mediation under interchange intervention.
