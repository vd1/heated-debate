## The Radical Alternative Worth Naming

> **"Morpho Collateral Transparency Standard (MCTS) — Sigma Labs as reference implementer of a public disclosure schema that Credora, curators, insurers, and other risk providers can adopt."**

The difference is significant:
- **v0.4 framing:** Sigma Labs is a vendor-substitute for Credora, selling badges instead of grades.
- **v0.5 framing:** Sigma Labs is the editor of a schema that *includes* Credora's outputs, curator manifests, insurer inputs, and anyone else's contributions. Sigma Labs operates the first canonical instance; others can fork.

This framing:
- ✅ Resolves the "indistinguishable from a data vendor" hollowing concern (§2): the moat is schema stewardship, not data

- ✅ Resolves the month-7 sustainability question (§5): whoever forks or federates the registry continues the work
- ✅ Resolves the moat/scope tradeoff (§2): Sigma Labs' value is the schema + reference implementation, not the dashboard
- ✅ Fits LlamaRisk's "protocol-owned infrastructure" wave (Document 4)
- ✅ Fits Prosperi's implicit argument (Document 5): mispricing persists because facts aren't shared; a shared fact-layer changes the equilibrium

**Cost:** one more draft iteration. **Benefit:** a proposal that wins on conviction, not just on modesty.

Agent A’s “differentiation collapse” premise is partly a category error.

A Morpho-funded disclosure layer should not need a proprietary moat. If the point is public collateral transparency, then forkability is a feature, not a defect. Governance should evaluate this like public infrastructure: credibility, reproducibility, neutrality, adoption path, and failure modes. “What stops someone copying this?” is the wrong primary question. The better question is: “If Sigma Labs disappears, does Morpho still own something useful?”

The moat, if any, should be procedural, not proprietary: schema stewardship, detector quality, reproducible history, community trust, and adoption by curators or other risk providers. Trying to preserve a commercial moat inside a public-good RFC reintroduces the exact vendor-risk problem the proposal is supposed to solve.

**Three Structurally Blocking Regressions**

1. **Conflict-of-interest disclosure absent.**  
This is the biggest blocker. If a team member curates, invests in, advises, or competes with covered vaults, the RFC must disclose it and define recusal rules. Without this, “facts-first” can still be attacked as selective disclosure.

2. **Day-1 open-source commitment absent.**  
For a public-good disclosure layer, “open-sourced upon delivery” is too weak. Governance is not buying a black-box vendor deliverable. The repo, schema, detector logic, issues, CI, and reproducibility harness need to be public from Month 1.

3. **No evidence of ecosystem demand.**  
The repeated absence of LOIs or named trial commitments is not a small omission anymore. At least one curator, risk provider, or data consumer should say: “we would test/use/review this if delivered.” Otherwise the manifest and labels look internally invented.

**Tactical Spec Items, Not RFC Blockers**

These belong in the delivery spec, acceptance criteria, or appendix:

- Signed registry snapshots
- Detector versioning and changelog
- Confidence enum
- Detection Gap Log
- `MCTI Schema v0.1` naming
- Deterministic top-30 methodology
- Covered asset/material incident definitions
- Acceptance criteria per deliverable
- Adoption milestone mechanics
- More detailed team bios

They matter, but they should not be treated as equal-severity governance blockers. Overloading the RFC with all of them risks turning a clean public-good ask into procurement sludge.

**Where Agent A Overreaches**

The “Dune + DeFiLlama” criticism is too dismissive. Dune shows queries. DeFiLlama shows aggregates. This proposal’s value is supposed to be normalized structural decomposition: dependency graphs, recursive exposure, oracle/admin facts, diff events, and vault-level labels. That is not automatically commodity.

But Agent A is right that the draft undersells the integration layer. The proposal should stop defending itself as a product and present itself as a disclosure standard plus reference implementation.

**Sharper Alternative**

Rename the object:

> Morpho Collateral Transparency Standard — with Sigma Labs building the first open reference implementation.

That makes the public-good logic coherent. Credora, curators, insurers, depositors, and governance can all consume the same fact layer. Sigma Labs does not need to be the moat. The schema, reproducibility, and adoption process are the moat.
