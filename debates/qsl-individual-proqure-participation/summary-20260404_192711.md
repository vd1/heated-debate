### Key convergence: Access-based participation framework

**The core principle:** The relevant control surface is decision influence and unfair supplier advantage, not employer label.

**Three participation classes (by access, not by org chart):**

**Key governance mechanisms:**
- Independent integrity panel (assurance body, not decision-maker — contracting authority retains final decision under ss.81-83)
- Written reasons required if contracting authority departs from panel recommendation
- Rebuttable presumption materiality test (no hard monetary thresholds)
- Auditable segregation: IT access logs, meeting records, roster freeze, methodology version control
- Attestation-based disclosure (role-based, not named rosters)
- Three-step escalation: mitigate → route-switch → exclude (mandatory if unfair advantage cannot be avoided)
- Lifecycle-anchored cooling-off periods (not arbitrary fixed months)

### Best ideas

1. **"Employer is the wrong control surface; access to evaluation functions is the right one"** — the one-liner that reframes everything for DSIT.
2. **Rebuttable presumption materiality test** — avoids gaming of hard thresholds while providing structure.
3. **Three-step escalation (mitigate → route-switch → exclude)** — mirrors the Act's own logic and prevents gaming while preserving the mandatory backstop.
4. **Phase 1 fallback: Permitted class only** — recovers majority of programme value with minimal governance complexity. Builds track record for Phase 2 expansion.
5. **Comparator parity argument** — quantum hub researchers are permitted despite equivalent government funding; QSL's only distinguishing feature is TTE role, which the framework directly addresses.
