# Would Open Collider's "collision" idea produce better heated-debate outcomes?

Review the proposition that injecting **structurally-distant domain capsules** into the architect/reviewer prompts of `heated-debate` — in the spirit of Open Collider's operationalization of Koestler's bisociation — would produce *better debate outcomes* than the current creativity-dial-alone setup.

## What Open Collider does (one paragraph)

Open Collider (CL-ML/open-collider) is a Claude Code skill / API pipeline for non-trivial idea generation. Its central move: before asking the model for ideas, *force* it to reason through a counter-intuitive principle from a structurally distant knowledge domain — a "collision." Pipeline is: generate distant domains → for each (reference text × domain) pair, generate ~20 ideas in an isolated context → score against user-defined axes → curate with human love/like/trash → next iteration biases domain selection (fresh / deepen / refresh) from feedback. Premise: direct prompting traps outputs in a "default-prompt basin"; only cross-matrix injection escapes it.

## What heated-debate currently does

A round-based adversarial loop between an architect and a reviewer. A creativity dial cools linearly from 5 (Wild — "explore radical alternatives, question the premise") to 1 (Precise — "converge; correctness only"). The dial text is *instructional only* — no external content is injected. Topics are intended to be review-framed (see `todo.md`); imperative topics cause agents to drift into implementation.

## What the debate must decide

1. **Define "better outcome" first.** Less basin-convergence between architect and reviewer? More genuinely novel proposals surviving to round 1 (Precise)? Faster convergence on good plans? Higher-quality minority reports? Be concrete — the debate is useless without an operationalization the architect and reviewer both accept.

2. **At which dial levels would collision help vs. hurt?** Plausibly: helpful at dial 5 (Wild), neutral at 3 (Balanced), actively harmful at 1 (Precise) where it would re-open settled ground. Argue the shape of the curve.

3. **Is bisociation redundant given the adversarial structure?** The architect/reviewer split is *already* a frame-collision (proposer vs. critic). Does adding a distant-domain capsule on top compound the effect, or does the second frame drown out the first?

4. **Concrete design.** If collision helps, what's the minimal patch? Options: (a) inject one distant-domain capsule into both agents on dial=5 only; (b) inject *different* domains into A and B to deliberately desynchronize their frames; (c) make the domain a per-round parameter and let the dial control its "distance"; (d) keep collision out of the dial entirely and put it in a separate `--collide` flag that runs before round 1 as a "framing pass."

5. **Failure modes.** Where does this break? Too-distant domains producing noise; domains the model already over-associates with the topic (no actual collision); architect and reviewer agreeing on the *domain* rather than on the *topic* (worst case: both agents writing about the metaphor instead of the problem).

6. **Falsification.** Propose one concrete A/B that would settle whether collision improves heated-debate outcomes on a representative topic (e.g., one of the existing `debates/` cases). What gets measured, on what topic, against what baseline?

## Out of scope

- Implementing the patch.
- Defending or attacking Koestler's broader theory of creativity.
- The Open Collider scoring/curation loop — heated-debate has no equivalent and that's a separate question.

Architect: propose a concrete, testable integration (or argue convincingly that collision shouldn't be integrated at all). Reviewer: be willing to call the whole proposal a category error if the adversarial structure already does the work bisociation claims to do.

**Define Better Outcome**
- Primary metric: **more high-quality options survive into the final precise round**, not just more wild ideas in round 1.
- Measure:
  - `Novel viable proposals`: distinct proposals that are both non-obvious and implementable.
  - `Survival rate`: ideas introduced in Wild/Creative rounds that remain in the final plan.
  - `Reviewer independence`: reviewer critiques are not merely restatements of architect assumptions.
  - `Convergence quality`: final output is concrete, bounded, and decisionful.
  - `Drift penalty`: subtract for metaphor addiction, off-topic theory, or reopened settled issues.
- “Better” should mean: **higher final-plan quality at equal or lower drift**, not “more creative transcript.”

**Why Collision Helps**
- The dial says: “be more creative.”
- A collision capsule says: “reason through this alien operating principle, then map it back.”
- That matters because LLMs often satisfy “be creative” by sampling nearby tropes:
  - add a judge
  - add scoring
  - add phases
  - add memory
  - add tournament mode
- A structurally distant capsule can force different proposal classes:
  - from ecology: niche competition, invasive species, succession
  - from aviation incident review: checklists, black boxes, near-miss reporting
  - from immune systems: self/non-self, inflammation, tolerance, memory
  - from court procedure: evidentiary standards, burden shifting, admissibility

**Dial Curve**
- **Dial 5: strongly helpful**
  - Best place for collision.
  - Goal is to break the default basin before the debate hardens around obvious architecture.
  - Collision should produce candidate frames, not decisions.
- **Dial 4: conditionally helpful**
  - Useful if round 1 was bland or converged too quickly.
  - Risk starts rising: the agents may keep inventing instead of refining.
- **Dial 3: mostly neutral**
  - Balanced rounds should compare frames, not introduce fresh alien ones.
  - Use only if a moderator detects sameness.
- **Dial 2: harmful**
  - Focused refinement and distant-domain reframing fight each other.
  - At this point collision mostly creates churn.
- **Dial 1: actively harmful**
  - Precise means closure.
  - Collision here reopens settled ground and rewards cleverness over correctness.

**Is Adversarial Debate Already Collision?**
- No, not enough.
- Architect/reviewer is a **role collision**, not a **conceptual collision**.
- The reviewer can still critique from inside the same default frame:
  - “add tests”
  - “watch edge cases”
  - “make flags explicit”
  - “avoid scope creep”
- That is useful, but it is not bisociation.
- However, stacking role-collision and domain-collision can overload the prompt.
- Therefore the capsule must be:
  - short
  - operational
  - explicitly mapped back to the topic
  - banned from becoming the subject of the debate

**Concrete Design**
- Best minimal design: **separate `--collide` framing pass before round 1**.
- Do not bake collision into the creativity dial.
- Reason:
  - the dial controls behavior across rounds
  - collision controls initial framing
  - mixing them makes attribution impossible

**Proposed Flow**
- `--collide` enabled:
  - Generate or select one structurally distant domain capsule.
  - Run a short “framing pass” before Agent A’s first proposal.
  - Inject distilled collision output into Agent A and Agent B round 1 prompts.
  - Remove it after round 1 unless explicitly referenced by surviving proposals.
- Capsule format:
  - `Domain`
  - `Counter-intuitive principle`
  - `Mapping challenge`
  - `Forbidden behavior: do not write about the metaphor; apply the principle to the debate topic`
- Example:
  - Domain: aviation accident investigation
  - Principle: systems improve more from near-miss reporting than blame assignment
  - Mapping: design heated-debate to capture almost-failures in reasoning, not just final conclusions

**More Radical Variant**
- Inject **different capsules into Architect and Reviewer**:
  - Architect gets ecology: diversity, succession, niche pressure.
  - Reviewer gets courtroom evidence: admissibility, burden of proof, exclusion rules.
- This deliberately desynchronizes frames.
- Benefit: less collusive agreement.
- Risk: agents talk past each other.
- Use only for Wild round experiments, not default behavior.

**Best Default Patch**
- Add `--collide=pre` mode.
- One shared capsule.
- Round 1 only.
- Dial remains unchanged.
- Final round explicitly says:
  - “Discard any collision-derived idea that did not prove useful.”
- This gives clean attribution and low implementation risk.

**Failure Modes**
- **Metaphor capture**
  - Agents debate aviation/ecology/law instead of heated-debate.
  - Mitigation: capsule must say “use the principle, not the metaphor.”
- **Fake distance**
  - Model picks domains already over-associated with software design: architecture, games, markets.
  - Mitigation: ban obvious domains.
- **Too much distance**
  - Quantum mechanics, mythology, or cuisine may produce decorative nonsense.
  - Mitigation: require each capsule to expose an operational principle.
- **Shared-frame collapse**
  - Both agents agree on the capsule and stop debating the actual topic.
  - Mitigation: reviewer prompt must critique both the proposal and the collision mapping.
- **Late-round contamination**
  - Clever new frames appear during convergence.
  - Mitigation: no collision below dial 4.

**Falsification Test**
- Topic: use `review-dials-refactor` or this exact Open Collider integration question.
- Baseline:
  - normal heated-debate, 5 rounds, creativity dial only.
- Variant:
  - same models, same rounds, same topic, `--collide=pre`, one round-1 shared capsule.
- Run at least 5 paired trials per condition if stochasticity is available.
- Blind judge compares final outputs only, not transcripts.
- Score each final plan on:
  - novelty of surviving proposals
  - feasibility
  - specificity
  - relevance
  - convergence quality
  - drift penalty
- Collision wins only if:
  - final-plan score improves by meaningful margin
  - drift does not increase materially
  - at least one collision-originated idea survives into the final plan

**Concrete Proposal**
- Integrate collision experimentally, not as a new default.
- Start with `--collide=pre`.
- Shared capsule, round 1 only.
- No collision at dial 3 or below.
- Add transcript metadata marking:
  - capsule domain
  - principle
  - which final proposals trace back to collision
- Promote to default only if A/B shows better final plans, not merely more interesting early rounds.

### Agent B · `gpt-5.5` · 28s

- **Main objection:** the proposal assumes the bottleneck is “default-prompt basin.” The repo evidence points elsewhere: topic framing, role drift, convergence management, and weak evaluation. Collision may make transcripts more interesting while making the actual debate protocol less reliable.

- **“Better outcome” is still too creativity-biased.**
  - “More high-quality options survive” sounds good, but survival is not necessarily quality. Bad metaphors can survive because they are memorable.
  - Better should be judged by final decision quality: fewer contradictions, clearer tradeoffs, implementable next steps, fewer reopened issues.
  - Novelty should be a secondary metric, not the target.

- **The adversarial structure is already doing the useful part of collision.**
  - Architect vs reviewer is not merely “role collision”; it is a pressure system.
  - If reviewer independence is weak, the fix is stronger reviewer instructions, burden-of-proof rules, or rotating critique modes.
  - Injecting ecology or aviation may mask the real failure: the reviewer is not being forced to attack assumptions rigorously enough.

- **The proposed `--collide=pre` pass is cleaner than dial injection, but still suspect.**
  - A shared capsule gives both agents the same alien frame, which can reduce disagreement rather than increase it.
  - It risks creating a new consensus basin: “we are all thinking through aviation accident review now.”
  - If the capsule is removed after round 1, attribution becomes messy: did it help, or did it merely seed vocabulary?

- **Different capsules for A and B is more interesting, but more dangerous.**
  - It may create genuine frame conflict.
  - It may also make the debate incoherent: architect optimizes for ecology, reviewer argues courtroom admissibility, and nobody resolves the product question.
  - This needs a moderator or judge that can translate between frames, otherwise it becomes parallel monologue.

- **Dial curve claim is plausible but too neat.**
  - Dial 5 collision can help, yes.
  - Dial 4 may already be too late if round 1 framed the issue badly.
  - Dial 3 collision could be useful as a rescue mechanism when both agents are converging on garbage.
  - Dial 1 should almost certainly ban it.

- **Bigger failure mode:** collision rewards cleverness exactly where heated-debate already risks drifting from review into invention.
  - The repo’s known issue is imperative topics causing implementation drift.
  - Collision may worsen that by giving agents permission to reframe instead of review.
  - Before adding distant domains, fix topic contract enforcement.

**Radical Alternative**

Do not inject domain capsules. Inject **failure capsules**.

Instead of “learn from aviation/ecology/law,” force agents through concrete adversarial test cases:

- “This proposal must work when the topic is badly framed as an imperative.”
- “This proposal must work when both agents agree too early.”
- “This proposal must work when reviewer critiques become generic.”
- “This proposal must work when final output is novel but unactionable.”

That is closer to the actual system failure. It preserves relevance while still breaking the default basin.

**Better Minimal Experiment**

Compare three variants, not two:

1. Baseline: current creativity dial.
2. Domain collision: `--collide=pre`.
3. Failure collision: pre-round adversarial failure scenario, no distant metaphor.

Judge only final outputs, blind to condition.

Metrics:

- final plan actionability
- number of unresolved contradictions
- specificity of implementation recommendations
- reviewer independence
- drift penalty
- novelty that directly improves the final plan

Collision only wins if it beats both baseline and failure-capsule mode. If failure capsules win, Open Collider-style bisociation is the wrong tool; the system needed sharper local adversarial pressure, not distant-domain injection.

**Improvement To Proposal**

Keep `--collide` experimental, but split it:

- `--collide=domain`: Open Collider-style distant capsule.
- `--collide=failure`: local adversarial scenario.
- `--collide=asymmetric`: different frames for A and B, Wild round only.

Default should remain off. The stronger hypothesis is not “distant domains improve debates.” It is: **structured disruption improves debates when it targets the actual failure mode.**
