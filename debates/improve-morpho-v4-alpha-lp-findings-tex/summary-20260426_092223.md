3. **Reorder sections** (in particular: insert `Main Findings and Failure Modes` after the abstract; move `Sandwich and JIT` later; move `Demand-Curve Approximation` to the appendix).

## Unresolved disagreements / loose ends

- **Edge-distance statistics not computed.** The agreed `d_edge ≤ max(120, 0.05 w_B)` threshold is a reporting convention, but neither agent ran it on the data — the actual median and range need to be computed before the convention is applied or it's a placeholder.
- **No source verification.** Final patch was drafted from the snapshot, not the live `.tex`. Surviving uses of `alpha inventory`, `alpha rebalances`, `alpha strategy` and any duplicate sections still need a real grep+fix pass.
- **Statistical apparatus debate.** A wanted a full null-model battery; B argued for three falsifiers only because N≈31. Settled near B's view but the question of whether that's enough rigor for a "candidate signal" claim was not closed.

## Best ideas (most actionable)

1. **Three-products framing** (round 1, B): split the document mentally into forensic reconstruction, behavioral model, and tradable signal. Treat each with different rigor standards. This is the single cleanest reframing.
2. **Mechanical "shoulder" test**: refuse the metaphor unless `d_edge` passes a stated threshold. Forces an actual measurement instead of visual reading.
3. **Negative-control pool / asset** (B, round 1): if the same Binance-drift trigger "explains" rebalances on an unrelated v4 LP, the model is overfit. This was raised but didn't make it into the final patch — worth adding.
4. **Time-ordering audit** (B, round 1): the Binance price used to "explain" a rebalance must be known before submission, not just before landing. Otherwise the trigger model uses post-decision information. Also dropped from the final patch.
5. **Address-clustering uncertainty** (B, round 1): "same executor" is weaker than "same entity" without calldata patterns / shared relayers / nonce behavior. Should be added as a failure mode.
