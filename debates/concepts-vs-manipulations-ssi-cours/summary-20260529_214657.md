# Debate summary — concepts vs concrete manipulations in the SSI cours

**Quality score: 8/10.** Strong convergence on a locked, runnable 30-min spec; the binary was productively dissolved into a third frame (diagnostic questions). Loses a point because the room being modeled is assumed cooperative and AI-curious — heterogeneous or skeptical execs are under-explored. Panel scored it 8.4 but they over-rate convergence as quality; mine accounts for the audience-variance blind spot.

## Key conclusions

- The "concepts vs manipulations" framing is a false binary. The synthesis: **four diagnostic questions as scaffold, one worked example as belief, pilot brief as the delegable transfer artifact.**
- The four questions, phrased as questions (not nouns or an acronym), are the conceptual spine and stay visible on screen from minute 0:30. SSI is never spoken aloud in the room — it lives only in the footer of the leave-behind.
- The two original canonical examples collapse to **one live example** (memo evolving into a saved workflow run on a second input). The recurring-workflow case is exiled to page 2 of the leave-behind.
- The Monday deliverable is one **filled pilot brief** with a named verifier and one supervised run — not "play with agents," not "answer questions on paper."

## Key disagreements (resolved)

- **Failure mode for the first attempt:** A wanted a sharp visible hallucination (wrong number); B argued for "plausible but unsignable" (under-sourced, no criteria). B won — sharp failure tips into "agents are dangerous" rather than "agents need discipline."
- **Order of concept vs demo:** A wanted concept named *after* a felt rupture; B wanted concept as a scorecard *before* the demo. Settled on B: questions are a diagnostic lens the room uses to score the demo.
- **Two examples or one:** A initially kept both; B forced consolidation to one live. Settled on one live, second example in the leave-behind.

## Best ideas

1. **The pilot brief is the unit of delegation** — task, owner, verifier, data boundary, sources, acceptance criteria. Six fields, four of which map onto the four questions. Introduced once at 14:00 as "the four questions written down for a team."
2. **"Which would you fix first?"** instead of "which question failed?" — turns the diagnostic into an executive prioritization act and avoids the room stalling on wrong-precision.
3. **Acceptance criteria drafted live before saving the workflow** (10:30) — gives the second-input run a benchmark and turns the later sign-off block into refinement rather than first creation.
4. **First output preloaded, second run live** — kills the biggest timing risk (4 min of waiting on round 1) without losing authenticity.
5. **Worked criteria examples in the leave-behind** (cites source, separates facts/assumptions/recommendations, flags weak source material, fits format, names what verifier checks) — prevents Monday briefs from collapsing into "good quality."

## Weaknesses

- **Audience variance is the unexplored gap.** The plan assumes execs cooperative enough to answer "which would you fix first?" and authoritative enough to assign a verifier Monday. Skeptical, novice, or non-decisional rooms aren't planned for.
- **Facilitator-sensitive.** Tight time boxes (3-min governance cap, 90-sec criteria capture) need real discipline. One slow exchange and the buffer vanishes.
- **One live example is a bet** that the memo case carries both the segmentation/memory lessons and the repository/packaging lessons. If the second-input run feels redundant, the repository idea doesn't land in the room and exists only on page 2.
- **The recurring-workflow case on page 2 risks being skipped.** Execs read page 1, photograph it, and never reach page 2 where the second canonical example lives.

## Locked spec (the actionable output)

**30-min run-of-show:**
1. `0:00-1:30` — show the four questions (no acronym).
2. `1:30-5:30` — prompt + preloaded plausible-but-unsignable output.
3. `5:30-10:30` — "Which would you fix first?" Apply segmentation + sources. Re-run a focused piece.
4. `10:30-12:00` — draft 3 acceptance criteria with the room.
5. `12:00-14:00` — save prompt + criteria as workflow. Name portability once.
6. `14:00-15:30` — reveal pilot brief.
7. `15:30-21:30` — room chooses between two pre-cleared inputs. Run live.
8. `21:30-24:30` — sign-off + governance (capped 3 min).
9. `24:30-27:30` — hand out leave-behind, give Monday instruction.
10. `27:30-30:00` — buffer (reserved).

**Leave-behind (2 pages):** Page 1 = four questions + memo recap + filled pilot brief + criteria examples box. Page 2 = recurring-workflow case + blank brief template + governance checklist + Monday instruction + SSI footer.

**Hard requirements:** backup screenshot of the first output; two pre-cleared second inputs; paper handout (no QR); if RGPD surfaces, route to follow-up.
