## Substance Report

**Settled Conclusions:**
- The Leg Ledger Graph is the operating standard for DeFi yield decomposition, not Framework E or any single-label taxonomy
- Framework B (Payer x Mechanism) is mandatory for cash-flow attribution; Framework D (risk-first/short-risk) is mandatory for stress attribution; Framework C (persistence/sustainability) is optional as an overlay
- Framework A (the 4-source folk taxonomy: Farming/Staking/Arb-fund/RWA) is rejected — it mixes ontological layers and fails on every case that matters
- Framework D is the runner-up: best standalone stress projection, but not a substitute for the full graph
- The base graph must store descriptive facts only; all contested judgments (dependency scoring, opacity haircuts, subsidy-adjusted yield, scenario vectors, position-limit logic) belong in a separate versioned analytics layer
- The holder-perspective leg is the unit of analysis, not the product or protocol
- The graph must allow 'no direct payer' explicitly for control-layer legs, redemption-priority legs, MTM transfer legs, and linked negative-transfer legs
- Gross cash-flow source and net-return drags must always be recorded separately — never collapsed
- LPs, MEV, options/insurance, and allocator products must be represented as multi-leg objects, not single-row summaries
- Carry failure and exit/unwind failure are distinct states and must be recorded separately
- 'unknown/mixed' is a correct classification output for opaque wrappers, OTC hedged structures, points-heavy systems, or weak provenance — not a model failure
- Wrappers and control layers are first-class risk objects when they shape realization more than the nominal yield source
- Common-shock tags must be sparse and standardized: exchange venue, collateral complex, governance actor, oracle stack, incentive regime, redemption queue
- Dependency links belong in the base graph as existence claims only; severity scoring belongs in the analytics layer
- The counterfactual-demand test is the anchor for Goodhart resistance: if demand cannot survive removal of incentives/rebates/points within one reporting cycle, analytics should default ultimate source to subsidy/reflexive unless disproven
- Evidence tier is required for every leg: observed onchain, contractually disclosed, inferred, unknown
- Weak evidence must propagate into analytics through higher opacity penalties and tighter sizing — not sit as metadata
- Vocabulary governance should use a frozen core plus extension namespace; annual updates are too slow, continuous updates destroy comparability
- The framework is strong for subsidy detection, dependency mapping, concentration, and contagion analysis — but it is explicitly not fraud detection
- The main portfolio error from bad analytics is false diversification, not just single-position misread
- Schema and analytics must be versioned separately to prevent silent relabeling of old risks under new narrative names

**Open Questions:**
- What is the optimal cadence for extension namespace ratification — quarterly was proposed but not stress-tested against real governance friction
- How should the counterfactual-demand test be operationalized in practice when incentive removal has never been observed for a given protocol
- What minimum coverage of the top-N products is required before the graph provides portfolio-level value rather than boutique research
- How should the analytics layer handle disagreement between risk teams on dependency scoring — what governance process resolves contested materiality assessments
- Can the common-shock tag set remain stable, or will DeFi primitives like restaking, intent solvers, and prediction markets require new shock categories
- How should the graph represent hidden rehypothecation or fabricated assets that are structurally invisible to taxonomy-based analysis
- What is the right format for scenario vectors in the analytics layer — discrete stress scenarios vs continuous factor models
- How should cross-chain dependency and bridge risk be represented in the common-shock framework
- What threshold of opacity or missing decomposition should trigger hard position limits vs soft haircuts

**Best Ideas:**
- Separation of base graph (descriptive facts) from versioned analytics layer (contested judgments) — this is the key architectural insight that makes the standard governable without pretending scoring is objective
- The counterfactual-demand test as the primary Goodhart-resistance mechanism: if yield disappears when incentives disappear, the source is subsidy regardless of who the immediate payer appears to be
- Treating 'unknown/mixed' as a feature rather than a failure — opaque products get escalated to tighter limits rather than forced into false precision
- Publishing subsidy-adjusted gross yield alongside stress-adjusted net yield as the two headline portfolio numbers
- Multi-leg decomposition that forces separation of fee income from transfer-loss legs (e.g., Uniswap v3 swap fees vs IL/adverse-selection), premium intake from tail-loss exposure (DOVs), and venue carry from wrapper control (allocator products)
- Common-shock mapping as the portfolio-aggregation mechanism — two positions can look diversified by mechanism while sharing exchange venue, collateral complex, governance actor, or incentive regime
- Frozen core plus extension namespace for vocabulary governance — balances stability/comparability against the pace of DeFi innovation
- Distinguishing carry failure from unwind/exit failure as separate risk states — many DeFi products break on exit liquidity, queue position, or routing discretion before headline cash flow visibly stops
- Evidence-tier tagging with mandatory propagation into sizing — weak provenance is not just a data quality note, it is a position-limit input
- The wrapper as first-class risk object — in allocator products like Resolv USR, redemption priority, routing discretion, and gating can dominate the economics of the underlying legs

**Dependency Chains:**
- The Leg Ledger Graph base object must exist before Framework D can function as a useful stress projection — D is a view of the graph, not a standalone tool
- Morpho vault lending demand depends on downstream farming/incentive economics — if those subsidies disappear, borrow demand collapses and the lending leg's 'productive' character is revealed as quasi-reflexive
- sUSDe yield depends on three separable upstream sources (perp funding market, protocol treasury, and potentially collateral-stack yield), each with different failure modes and common shocks
- MEV value flows through a three-leg chain: user-loss → searcher-profit → validator-capture, where each leg has different risk characteristics and the same cash flow occupies different graph positions depending on perspective
- Resolv USR carry depends on downstream venue economics, but realization depends on wrapper/allocator behavior — the wrapper-control leg can dominate the venue-carry leg during stress
- The analytics layer depends on the base graph being stable and descriptive — if contested judgments leak into the base object, governance collapses into fights over model outputs
- Vocabulary governance depends on the frozen-core/extension-namespace architecture — without it, silent relabeling of old risks as new narratives undermines the entire comparability promise
- Portfolio-level false diversification detection depends on common-shock tagging being standardized and sparse — free-text tags break aggregation immediately
- The adoption path depends on manual pilot coverage of transparent flagship products first, with opaque products escalated to unknown/mixed rather than blocking the rollout

**Concrete Proposals:**
- Build the Leg Ledger Graph with these per-leg fields: immediate payer, ultimate source, mechanism (from frozen set), primary short-risk, secondary risks, gross cash-flow source, main net-return drags, break condition, unwind condition, wrapper/control facts, dependency links (existence only), common-shock tags (from standardized set), and evidence tier
- Implement the analytics layer as a separate versioned object storing: mechanical dependency assessment, economic materiality score, base/stress dependency state, opacity haircut, subsidy-adjusted yield, scenario vectors, and position-limit logic
- Apply the counterfactual-demand test: if demand cannot survive removal of incentives/rebates/points within one reporting cycle, default ultimate source to subsidy/reflexive unless disproven
- Publish four views from the same graph object: funding-source view (sustainability/subsidy dependence), stress view (portfolio stress map), dependency view (contagion/reflexivity), and common-shock view (hidden concentration)
- Publish two headline portfolio numbers: subsidy-adjusted gross yield and stress-adjusted net yield
- Start with manual pilot on top-50 TVL/yield transparent products using docs, contract mechanics, and onchain cash-flow observation
- Use the 8-step analyst rulebook: (1) split by leg, (2) allow no-direct-payer, (3) record payer structure with counterfactual test, (4) separate gross/net, (5) record carry and unwind failure, (6) tag common shocks from standardized set, (7) default to unknown/mixed for opaque products, (8) state framework limitations plainly
- Govern vocabulary with frozen core plus extension namespace, with quarterly extension ratification cycles
- Treat missing decomposition as a red flag requiring position limits, not a temporary data gap
