The Idea

Good design emerges from structured disagreement

🏗️

Agent A — Architect

Proposes implementation plans. Refines based on feedback. Sends diffs, not full repeats.

🔍

Agent B — Reviewer

Challenges proposals, finds flaws, identifies edge cases. Forces the plan to earn its shape.

Both agents run in the same model or across models. No shared context — each sees only what the other sends.

Creativity Dial

The subscription CLIs don't expose temperature. The dial is the only behavioral modulator — and it's the trainable signal for RL later.

Architecture

shelley.ts — the Bun engine

🦩 Claude backend

Session continuity via --resume <session-id>. New sessions with --session-id + --system-prompt.

⚡ Codex backend

Stateless calls via codex exec --sandbox read-only. Conversation history windowed manually per-agent.

🔐 Env sanitation

Strips provider API keys before spawning CLIs. You stay on subscription auth — no accidental billing.

# Run 5 rounds, default topic bun shelley.ts 5 # Topic from file, 3 rounds bun shelley.ts -f debates/stock-trades/topic.md 3 # Switch both agents to Codex MODEL_A=codex:gpt-5 MODEL_B=codex:gpt-5 \ bun shelley.ts -f debates/my-topic/topic.md 5 # Resume a partial run RESUME_FROM_LOG=debates/my-topic/20260227_1930.md \ bun shelley.ts -f debates/my-topic/topic.md 7
Context injection

Put a context.txt next to your topic.md listing repo-relative file paths. shelley.ts reads them fresh every run — no copies, agents always see current source.

Model Support

Any two models. Any combination.

☁️

Claude (claude:*)

Persistent sessions via --resume. haiku, sonnet, opus. No API key — subscription login.

⚡

Codex (codex:*)

Stateless codex exec. History windowed per-agent. Reasoning effort tunable via env var.

🌐

Any via LiteLLM

later/pynaille.py routes to Gemini, GPT-4o, and any LiteLLM provider with API keys.

# Claude vs Claude bun shelley.ts -f debates/perf/topic.md 5 # Codex vs Codex, high reasoning MODEL_A=codex MODEL_B=codex CODEX_REASONING_EFFORT=high \ bun shelley.ts -f debates/perf/topic.md 5 # Mixed: Claude architect vs Codex reviewer MODEL_A=sonnet MODEL_B=codex bun shelley.ts -f debates/perf/topic.md 5
What Comes Next

From debates to design machines

1

Dial vector

Expand from one scalar to a vector: creativity, risk tolerance, verbosity, deference — per agent, per round.

2

Phase architecture

Brainstorm → Hard spec → Scaffold → Tests → Fill-in. Each phase in its own conversation; output feeds next.

3

Tournament + judge

Run N parallel debates, different dial settings or model pairings. A judge agent scores output. Winner advances.

4

RL over dial vectors

Treat the dial vector as the action space; judge score is reward. Bayesian opt / evolutionary strategies — no deep RL infra needed.

5

Agent SDK path

With an API key, switch to the Claude Agent SDK for persistent in-process agents — eliminates the ~50s cold-start per turn.

6

Token accounting

Add token counts per turn to transcripts; feed into cost-aware dial optimization.
